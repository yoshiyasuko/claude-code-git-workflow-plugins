プロジェクトのライフサイクルフック（`.claude/skill-hooks.md`）を対話的にセットアップする。フックポイントにスキルを紐付け、生成結果をプレビューしてからファイルに書き出す。「フックを設定して」「skill-hooks を作って」「フックをセットアップして」などのリクエストで使用する。

## 手順

### 1. プロジェクトのスキルを検出

以下のディレクトリから `.md` ファイルを Glob ツールで検索し、利用可能なスキルの一覧を作成する：
- `.claude/commands/*.md`
- `.claude/skills/*.md`
- `.agents/skills/*.md`

ディレクトリが存在しない場合は無視する。検出されたスキルはファイル名（拡張子なし）をスキル名とする。

スキルが1つも検出されなかった場合、AskUserQuestionツールで確認する：
- 質問文: 「プロジェクトにスキルが見つかりませんでした。スキル名を手動で入力して続行しますか？」
- 選択肢: ["手動入力で続行", "処理を中止"]
- 「処理を中止」→ 終了する

### 2. 既存の skill-hooks.md を確認

プロジェクトルートの `.claude/skill-hooks.md` を Read ツールで読み込む。

- **ファイルが存在する場合** → 現在の内容を表示し、「既存の設定に追記・編集します」と伝える。既存のフック定義をパースして保持しておく。
- **ファイルが存在しない場合** → 「新規作成します」と伝える。

### 3. フックポイントの設定

以下の3つのフックポイントについて、順番に設定を行う：

1. **pre-commit**（commit コマンド：ステージング前に実行）
2. **post-push**（commit コマンド：プッシュ後にユーザー確認付きで実行）
3. **post-pr**（create-pr コマンド：PR 作成・更新後にユーザー確認付きで実行）

各フックポイントについて、AskUserQuestionツールで確認する：
- 質問文: 「`<フックポイント名>` フックに紐付けるスキルを選択してください（<フックの説明>）」
- 選択肢: 検出されたスキル名の一覧 + "スキップ（設定しない）"
- 既存の設定がある場合は、現在設定されているスキル名を質問文に含める（例: 「現在の設定: `<スキル名>`」）

スキルが選択された場合（「スキップ」以外）、続けて説明文を設定する：

- **pre-commit の場合**: 説明は不要（pre-commit は確認なしで自動実行されるため）。説明列はスキルの用途を簡潔に記述する（例: 「コミット前にlintを実行」）。AskUserQuestionツールで確認する：
  - 質問文: 「このフックの説明を入力してください（例: コミット前にlintを実行）」
  - 自由入力

- **post-push / post-pr の場合**: 実行時にユーザーに確認を求めるため、質問文と選択肢を設定する。AskUserQuestionツールで2回確認する：
  - 1回目 — 質問文: 「フック実行時にユーザーに表示する質問文を入力してください（例: デプロイスクリプトを実行しますか？）」、自由入力
  - 2回目 — 質問文: 「肯定的な選択肢のラベルを入力してください（例: 実行する）」、自由入力
  - 説明列の値は「質問: "<質問文>", 選択肢: ["<肯定ラベル>", "スキップ"]」の形式で組み立てる

### 4. プレビューの表示

設定内容から `skill-hooks.md` の全文を組み立て、AskUserQuestionツールのプレビュー機能で表示する：
- 質問文: 「以下の内容で `.claude/skill-hooks.md` を書き出しますか？」
- 選択肢（プレビュー付き）:
  - "書き出す" — プレビューに生成した skill-hooks.md の全文を表示
  - "やり直す" — 説明: ステップ3からやり直す

「やり直す」が選択された場合はステップ3に戻る。

生成するファイルのフォーマット：

```markdown
# Skill Hooks

このファイルは [git-workflow プラグイン](https://github.com/yoshiyasuko/claude-code-git-workflow-plugins) のライフサイクルフック設定です。`/commit` や `/create-pr` などのコマンド実行時に、各フックポイントで指定したスキルが自動的に呼び出されます。

## commit

| フック | スキル | 説明 |
|-------|-------|------|
| pre-commit | <スキル名> | <説明> |
| post-push | <スキル名> | <説明> |

## create-pr

| フック | スキル | 説明 |
|-------|-------|------|
| post-pr | <スキル名> | <説明> |
```

設定されなかったフックポイントの行は含めない。あるセクション（commit / create-pr）に該当するフックが1つもない場合、そのセクション自体を省略する。既存ファイルの編集時は、変更対象のフックのみ上書きし、既存の他のフック定義は保持する。冒頭の git-workflow プラグイン利用の説明文は、新規作成時には必ず含める。既存ファイルに同等の説明文がない場合は追記する。

### 5. ファイルの書き出し

`.claude/` ディレクトリが存在しない場合は作成する。Write ツールで `.claude/skill-hooks.md` を書き出す。

### 6. 結果の表示

- 作成・更新したファイルパス
- 設定したフックポイントの一覧（フック名 → スキル名）
